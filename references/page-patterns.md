# Page Patterns

Use these defaults when the user asks for a prototype that should feel like an internal enterprise system but does not specify the full page structure.

## List Page

Default sections:

1. Title and primary action
2. Filter panel
3. Secondary actions or batch actions
4. Data table
5. Pagination

Typical filters:

- keyword
- business type
- status
- owner
- date range

Typical table columns:

- name
- code
- category
- status
- owner
- updated time
- actions

Prompt snippet:

```text
Create a desktop admin list page with a compact filter panel on top, a restrained action bar, a bordered data table, and pagination. Keep the styling neutral and operational.
```

## Detail Page

Default sections:

1. Title, status, and actions
2. Summary card
3. Basic information block
4. Related records or tabs
5. Operation log or remarks

Prompt snippet:

```text
Create a desktop admin detail page with a status-aware header, a concise summary area, sectioned information blocks, and related records organized in tabs.
```

## Create Or Edit Page

Default sections:

1. Title and context
2. Form sections grouped by topic
3. Inline validation or helper text
4. Footer actions with a clear primary button

Prompt snippet:

```text
Create a desktop admin form page with grouped sections, compact labels and controls, practical helper text, and a stable footer action area.
```

## Drawer Flow

Use for:

- quick create
- quick edit
- side-panel inspection

Prompt snippet:

```text
Create a right-side drawer for a desktop admin system. Keep the layout compact, sectioned, and task-focused, with clear primary and secondary actions.
```

## Dashboard

Use cautiously. Keep it operational.

Default sections:

1. Title and global filters
2. Small summary cards
3. Supporting trend or distribution chart
4. Operational table or alert list

Prompt snippet:

```text
Create an operational dashboard for a desktop admin system with restrained metric cards, clear filters, and a supporting data table. Avoid flashy presentation design.
```

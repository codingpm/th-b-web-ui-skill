# Publishing Kit

Use this file as source material when publishing the skill to a skills platform, writing a catalog entry, or preparing example prompts.

## One-Line Description

Generate enterprise UI prototypes that feel like an internal component system, with structured admin layouts, standardized page recipes, and internal-style naming semantics.

## Short Description

Create desktop admin prototypes that follow internal UI patterns such as filter-panel-plus-table layouts, drawer workflows, grouped forms, standardized action bars, and business-aware selectors.

## Longer Description

This skill helps Codex generate prototypes, wireframes, Figma prompts, and frontend code that look and behave like a mature internal enterprise system. The skill encodes common internal page structures, component-family semantics, and UI writing habits so the output feels closer to real operational products and less like a generic consumer interface.

## Key Selling Points

- Use internal-style naming semantics such as base components, business components, and standardized blocks as design language
- Favor structured desktop admin layouts over marketing-style compositions
- Reuse standard enterprise page recipes for list pages, detail pages, forms, drawers, and dashboards
- Prefer business-aware selectors, status tags, fixed action areas, and standardized table regions
- Work for design briefs, Figma prompts, HTML, Vue, React, and other prototype outputs

## Best Fit Scenarios

- Generate a management list page
- Create a configuration or approval form
- Design a detail page with tabs and status
- Build a quick-create drawer workflow
- Draft a dashboard that feels operational rather than flashy
- Convert vague business requirements into internal-style page structure

## Positioning Notes

When presenting this skill publicly:

- describe it as an enterprise admin UI skill
- describe the design language as internal-style naming semantics for base components, business components, and standardized blocks
- emphasize that it captures page patterns and design language
- avoid framing the skill as tied to a specific unpublished technical implementation
